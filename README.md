# AeroSpace AutoKill

## Проблема

В MacOs если приложение закрывать не через cmd+q(например на красный кружок в titlebar, или exit в терминале) - у тебя остается запущенный процесс, который висит бесконечно долго.

Особенно это заметно если запускаешь приложения через skhd, например

```
alt - return : open -n /Applications/Nix\ Apps/Rio.app
```

За рабочий день может остаться 10-ки процессов, у которых нету окон.


## Решение

Написать программу, которая:

1. Получает список открытых окон 

```bash
aerospace list-windows --all --format "%{app-pid}:%{app-name}"
```

2. Получает список запущенных программ:

```bash
ps aux | grep Rio.app | grep -v grep

```

3. Проверят по PID у каких процессов нету окон и убивает их
